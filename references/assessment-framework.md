# AI时代产品经理能力评估体系

## 能力模型

评估采用“三层八维”模型：战略脑、洞察眼、工程骨、安全盾。本评估不固定题量，也不以单选题为主；它围绕八个维度逐步收集证据，并通过场景挑战、专家追问、变量扰动和失败复盘，形成一次结构化专家诊断，识别用户在AI时代做互联网产品时最可能放大产出的长板和限制上限的短板。

| 能力簇 | 维度 | 权重 | 招聘官观察重点 |
|---|---|---:|---|
| 战略脑 | A. 问题定义与机会判断 | 15% | 能否识别真实机会并定义愿景；把业务痛点转化为AI可解决的具体任务，而不是技术炫技 |
| 洞察眼 | B. 用户洞察与场景拆解 | 12% | 能否洞察市场、竞品、用户动机、使用频次、替代方案和关键阻力 |
| 工程骨 | C. AI能力理解与产品化 | 18% | 能否理解模型边界、数据/上下文/工具调用、幻觉、延迟、成本、评测和可靠性 |
| 洞察眼 | D. AI原生体验与工作流设计 | 12% | 能否平衡AI能力展示与用户认知负荷，设计可信、可控、容错的人机协作体验 |
| 工程骨 | E. 数据指标与实验决策 | 13% | 能否设计指标、评测集、数据采集、标注闭环、实验方案和因果判断 |
| 战略脑 | F. 商业化与增长策略 | 10% | 能否连接用户价值、获客、定价、留存、单位经济模型，并平衡推理/算力成本与商业回报 |
| 战略脑 | G. 优先级与资源取舍 | 10% | 能否在不确定性、资源约束、技术风险和商业收益之间协调算法、工程、数据、业务团队 |
| 安全盾 | H. 组织协同、伦理与风险 | 10% | 能否推动跨职能协同，并识别隐私、版权、合规、偏见、安全、品牌和责任边界风险 |

## 背景与目标岗位校准

同样的得分在不同背景下含义不同。评估必须结合用户当前职责和目标岗位解释：

- 初级/转型PM：重点判断可迁移产品基本功、AI产品化意识和学习路径，不轻易给“不能胜任”的绝对结论
- 高级PM：重点判断是否能独立负责AI功能或业务模块，是否能建立指标、评测、风险和迭代闭环
- 产品负责人：重点判断机会定义、资源取舍、商业闭环、跨团队推进和长期系统能力
- 垂直行业PM：问诊场景和报告必须贴近该行业约束，例如ToB交付、增长实验、内容安全、交易履约、金融/医疗合规等

报告中的“胜任度”不是单纯总分排名，而是“当前证据相对于目标岗位要求”的判断。

## 行业与产品形态加权

根据用户背景调整取证密度，而不是改变全部框架：

- AI Agent / 大模型应用：提高 C、D、H 的取证密度，重点看工具调用、长链路失败、评测、权限、人类确认点和可控性
- ToB / 企业服务：提高 G、H、F 的取证密度，重点看交付边界、客户成功、销售承诺、数据隔离、权限、合规、续费和组织推进
- 增长产品：提高 E、F 的取证密度，重点看实验设计、归因、渠道质量、反作弊、留存和单位经济模型
- 0-1产品：提高 A、B、C、G 的取证密度，重点看机会定义、用户证据、技术可行性、最小切入点和学习速度
- 高风险行业（金融/医疗/教育/未成年人）：提高 H、C、E 的取证密度，重点看责任边界、专家审核、误导风险、数据治理和安全兜底
- 资深/负责人目标：更多难度4-5的战略取舍题，要求说明组织机制和长期系统影响
- 初级/转岗目标：更多难度2-3的基础判断题，重点评估可迁移能力和学习路径

## 成熟度等级

| 等级 | 分数区间 | 描述 |
|---|---:|---|
| L1 功能执行型 | 0-39 | 主要关注需求接收和功能交付，容易把AI当作卖点或工具堆叠 |
| L2 场景分析型 | 40-54 | 能拆场景和用户问题，但在AI能力边界、指标和商业取舍上不稳定 |
| L3 业务闭环型 | 55-69 | 能围绕用户价值、数据指标和资源约束做较完整判断，AI产品化仍有盲区 |
| L4 AI原生产品型 | 70-84 | 能把AI能力、体验、评测、成本和业务结果连成闭环，具备独立负责AI产品模块的能力 |
| L5 战略负责人型 | 85-100 | 能在不确定技术周期中定义机会、组织资源、建立评测体系并推动业务增长 |

## 单轮证据评分

每轮场景问诊按0-3分内部评分；如果使用单选校准题，也映射到同一套标准。不要向用户展示分数。

- 3分：高成熟度判断。能兼顾用户价值、业务目标、数据证据、AI边界、风险兜底和组织落地
- 2分：可接受判断。方向基本正确，但遗漏关键约束、评估方法、成本风险或长期影响
- 1分：初级判断。抓住表层问题，但偏功能化、短期化、Demo化或缺少验证
- 0分：高风险判断。明显忽视用户、业务、技术风险、数据证据、合规伦理或责任边界

根据用户追问表现微调：

- 解释补齐关键约束：+0.5
- 解释暴露严重误区：-0.5
- 能在变量扰动后稳定修正方案：+0.5或提高置信度
- 面对失败复盘仍停留在“继续优化功能/模型”：不加分，并记录短板信号
- 最高不超过3分，最低不低于0分

每轮证据要拆成四类记录：

- 直接证据：用户对当前场景的核心判断
- 追问证据：用户对决策依据、变量扰动、失败复盘的回答
- 相邻证据：其他维度回答对当前维度的侧面支持或冲突
- 反面证据：用户后续表现修正了早期短板，或早期强项在压力下不稳定

## 难度加权

场景难度为1-5：

| 难度 | 场景特征 | 权重 |
|---|---|---:|
| 1 | 基础概念识别，单一变量 | 0.8 |
| 2 | 常见产品场景，少量取舍 | 0.9 |
| 3 | 中等复杂度，用户/业务/技术三者取舍 | 1.0 |
| 4 | 高不确定AI产品场景，多团队约束 | 1.1 |
| 5 | 战略级或负责人级判断，长期系统影响 | 1.2 |

单轮贡献 = 证据分 / 3 * 难度权重。最终按各维度累计贡献和权重换算到100分。

## 维度覆盖与取证建议

默认必须覆盖全部八个维度：

1. A 问题定义与机会判断
2. B 用户洞察与场景拆解
3. C AI能力理解与产品化
4. D AI原生体验与工作流设计
5. E 数据指标与实验决策
6. F 商业化与增长策略
7. G 优先级与资源取舍
8. H 组织协同、伦理与风险

推荐节奏：

- 每个维度先用1个贴近用户背景的开放式场景挑战取证
- 用户回答和追问已经能支撑判断时，切换到下一个维度
- 用户回答过泛、证据矛盾或推导不足时，用变量扰动、失败复盘或相邻场景补证
- 关键短板和目标岗位门槛维度需要更高置信度，必要时复测
- 最后一轮优先补齐 `needs_recheck` 或低置信维度，而不是机械出综合题

停止条件：

- 所有八个维度都有可引用证据，并且能够给出高/中/低置信判断
- Top优势和Top短板都有具体轮次、用户原始判断或追问证据支撑
- 关键短板不是单点失误，而是重复信号或在压力追问下稳定暴露
- 如果某些维度经复测仍不清晰，可以停止问诊，但必须在报告中标注低置信和待验证假设

## 短板与Derailer判断

短板不是单题答错，而是以下信号重复出现：

- 直接跳方案，缺少问题定义
- 把AI当功能包装，忽略模型边界、评测和运营成本
- 只看体验亮点，不看指标、留存和商业闭环
- 只追求自动化，忽略用户控制感、纠错和信任建立
- 用单一指标判断成败，忽略反指标和样本偏差
- 优先级判断依赖主观偏好，缺少收益、成本、风险和学习价值
- 忽略隐私、版权、合规、偏见、品牌伤害等AI产品风险

Derailer 是会显著影响目标岗位胜任的系统性风险，必须在报告中单列。常见 Derailer：

- 负责人目标下，无法把机会、资源、指标和组织动作连成闭环
- AI产品目标下，持续忽略评测集、失败兜底、成本、延迟和可信体验
- 高风险行业下，轻视隐私、合规、责任边界或专家审核
- 增长目标下，把增长等同于获客或转化，缺少留存、因果和单位经济模型
- ToB目标下，混淆客户定制、可复制产品能力和销售承诺

Derailer 需要至少2个证据点，或1个极高风险证据点加1个相邻维度支持。没有足够证据时写成“待验证风险”。

## 元能力观察

除八维能力外，记录两个元能力，但不要替代维度评分：

- 逻辑稳定性：面对追问、变量扰动和反事实时，判断框架是否稳定
- 学习敏锐度：被指出约束缺失后，能否快速吸收并修正方案

元能力用于解释“同分不同质”：同样70分，能在追问中修正的人比只背标准框架的人更值得培养。

## 有效性边界

这套评估不是权威认证，也不能替代真实项目复盘、作品集审查、面试深挖或绩效数据。它的价值在于：

- 用统一能力模型降低“凭感觉评价”的随意性
- 用真实场景挑战观察产品判断，而不是泛泛考AI概念
- 用背景校准和专家追问减少单选题误判
- 用证据化报告输出可复查的优势、短板、Derailer和待验证假设

最终结论应表述为“基于本轮证据的判断”，不要表述为不可推翻的人才定级。
